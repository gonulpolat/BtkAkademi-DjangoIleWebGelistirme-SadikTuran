* Web geliştirme frameworkü
* Server tarafında çalışır
* Hazır yönetim paneli var
* Alternatifleri => ASP.NET, PHP, Node.js

Komut İstemi
>>> python --version                     : kurulu olan Python versiyonunu gösterir
>>> django-admin                         : komutları listeler
>>> pip freze                            : kurulu olan kütüphaneleri listeler
>>> django-admin startproject courseapp  : proje oluşturur
>>> python manage.py runserver           : projeyi çalıştırır
>>> python manage.py startapp courses    : proje içine uygulama eklenir
>>> python manage.py makemigrations      : modelde yapılan değişiklikler için migration dosyası oluşturur
>>> python manage.py migrate             : migration'ları veri tabanına ekler
>>> python manage.py createsuperuser     : superuser oluşturur
>>> python manage.py collectstatic       : tüm static dosyaların setting içerisinde belirtilen isimdeki dosyada birleştirir

* Eklenen uygulamaları INSTALLED_APPS içerisine ekle

* Html sayfaları Django içinde template olarak adlandırılır

* VSCode içerisinde Django eklentisi olduğu için html emmet eklentisi çalışmıyor. Bunun için settings.json dosyası içerisine aşağıdaki özelliği ekle;
"emmet.includeLanguages": {
    "django-html": "html"
}

* Uygulamalar içerisindeki templates klasörü taranan klasör iken, ana dizine eklediğin templates klasörünü projeye tanıtman gerekli. Aynı durum static klasörü için de geçerli.


Layout
- layout.html       : Değişiklik yapılacak yerlere block eklenir
- içerik sayfaları  : {% extends "layout.html" %} sonra değişiklik yapılacak yerlere block eklenir. partial ayrılan yerler include ile birleştirilir.
- partial sayfalar  : sade html sayfası. herhangi bir şey eklenmiyor.

* partials == includes

null  -> veri tabanı ile ilgili
blank -> formlar ile ilgili
